BFC(Block formatting context)

块格式化上下文(MDN)

BFC定义：

BFC(Block formatting context)直译为"块级格式化上下文"。它是一个独立的渲染区域，只有Block-level box参与， 它规定了内部的Block-level Box如何布局，并且与这个区域外部毫不相干。

创建BFC的方式：

  • 根元素或其它包含它的元素
  • float 不为 none
  • position 为 absolute 或 fixed
  • display: inline-block, table-cell, table-caption, flex, inline-flex
  • overflow 的值不为 visible的元素

BFC规则：

  • 内部的Box会在垂直方向，一个接一个地放置。
  • Box垂直方向的距离由margin决定。属于同一个BFC的两个相邻Box的margin会发生重叠
  • 每个元素的margin box的左边， 与包含块border box的左边相接触(对于从左往右的格式化，否则相反)。即使存在浮动也是如此。
  • BFC的区域不会与float box重叠。
  • BFC就是页面上的一个隔离的独立容器，容器里面的子元素不会影响到外面的元素。反之也如此。
  • 计算BFC的高度时，浮动元素也参与计算

自适应两栏布局

BFC的区域不会与float box重叠。
.container-BFC1 {
  position: relative;
  width: 300px;
}

.container-BFC1 .aside {
  float: left;
  width: 100px;
  height: 150px;
  background: #f66;
}

.container-BFC1 .main {
  height: 200px;
  background: #fcc;
  overflow: hidden;
}

清除内部浮动

计算BFC的高度时，浮动元素也参与计算 
.container-BFC2 {
  border: 5px solid #fcc;
  overflow: hidden;
}

.container-BFC2 .child {
  float: left;
  border: 5px solid #f66;
  width: 100px;
  height: 100px;
}

防止垂直 margin 重叠

Box垂直方向的距离由margin决定。属于同一个BFC的两个相邻Box的margin会发生重叠

.container-BFC3 .wrap {
  overflow: hidden;
  /* child外面包裹一层容器，
    并触发该容器生成一个BFC。
    那么两个child便不属于同一个BFC，
    就不会发生margin重叠了。*/
}

.container-BFC3 .child {
  width: 200px;
  height: 100px;
  margin: 50px;
  background: #fcc;
}
